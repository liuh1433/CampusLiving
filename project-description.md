# 校园社交系统 — 项目描述与第一阶段规划

## 一句话介绍

一款基于**真实校园空间的匿名社交产品**。用户以虚拟身份进入 3D 校园场景，点击具体建筑、楼层和房间，在真实空间语境中「落座」、聊天、吐槽和共鸣互动。

当前第一阶段不做完整校园，先以 **河南大学金明校区综合教学楼建筑群（1-6 号教学楼）** 作为 MVP 场景，验证「建筑点击 → 楼层选择 → 房间/座位互动 → 空间聊天」的核心体验。

---

## 当前 MVP 范围

### 阶段目标

第一阶段只围绕综合教学楼开发，完成一个可交互、可扩展的 3D 建筑原型：

1. 用户进入综合教学楼区域地图。
2. 用户点击具体楼栋，例如 `1号教学楼`、`2号教学楼`、`综合教学楼 / 4号`。
3. 进入该楼栋的楼层选择界面。
4. 楼层选择时切换为**平视 + 透视图**，展示该楼栋每一层的 3D 楼层模型。
5. 用户点击某一层后进入房间列表或房间场景。
6. 后续接入座位系统和匿名聊天系统。

## 产品核心定位

| 维度 | 描述 |
|---|---|
| 产品类型 | 校园场景化匿名社交 |
| 目标用户 | 在校大学生，尤其是经常出入教学楼、自习室、公共教室的学生 |
| 核心价值 | 在真实校园空间中释放压力、寻找共鸣、偶遇同空间的人 |
| 差异化 | 以「空间」为社交载体，而不是以「人」为社交载体。用户不是先找朋友，而是先进入一个熟悉地点 |
| 第一阶段场景 | 综合教学楼 1-6 号教学楼 |

## 视觉与建模风格

### 总体风格

- Low-Poly 卡通校园风格。
- 色彩明亮、干净，参考校园地图的青绿色底盘、浅色道路、灰蓝屋顶、粉白建筑体块。
- 建筑不追求照片级真实，而追求**可识别的校园空间关系**。
- 视角分两类：
  - **建筑群地图视角**：正交等距俯视，类似校园导览图和模拟经营游戏地图。
  - **楼层选择视角**：平视 + 透视图，能看清每层楼板、走廊、教室分区。

### 建筑群地图视角

用于展示综合教学楼整体方位：

- 相机：正交相机。
- 角度：高位俯视，偏 45° 等距地图感。
- 用途：用户点击楼栋。
- 当前布局：
  - `1号教学楼`：独立位于左下。
  - `2号教学楼`：位于右侧主楼下部，横向伸出。
  - `3号教学楼`：位于中部，横向伸出。
  - `5号教学楼`：位于中上部，横向伸出。
  - `6号教学楼`：位于顶部，横向伸出。
  - `综合教学楼 / 4号教学楼`：右侧纵向主楼，连接 2/3/5/6 号楼。
  - 左侧保留 `马可广场` 与 `孔子像` 方位提示。

### 楼层选择视角

用户点击某栋楼后进入：

- 相机：透视相机。
- 角度：平视略俯，类似看建筑剖面或楼层沙盘。
- 展示方式：
  - 当前选中楼栋从建筑群中突出显示。
  - 其他楼栋弱化、半透明或隐藏。
  - 楼栋按楼层拆分为可点击层板。
  - 楼层之间留出轻微垂直间距，形成「爆炸楼层视图」。
  - 鼠标悬停某层时，该层高亮并显示楼层标签、在线人数、可进入房间数量。

### 楼层模型风格

每层楼不需要完整室内精装，第一版使用简化模型：

- 楼板：浅灰或暖白色矩形板。
- 外墙：低矮半墙或透明蓝色轮廓墙。
- 走廊：浅米色条带。
- 教室：用不同色块或半墙分区表示。
- 楼梯/电梯：用小方块或竖向核心筒表示。
- 门窗：蓝色扁平块。
- 文字标签：`1F`、`2F`、`自习区`、`大教室` 等。

## 楼栋与楼层规划

### 1号教学楼

- 位置：左下独立楼栋。
- 建议楼层：4 层。
- 房间类型：普通教室、自习教室、小组讨论室。
- 交互重点：作为第一栋可点击楼栋，优先实现完整流程。

### 2号教学楼

- 位置：右侧综合楼下部横向楼体。
- 建议楼层：4 层。
- 房间类型：大教室、公共课教室、自习区。

### 3号教学楼

- 位置：右侧综合楼中部横向楼体。
- 建议楼层：4 层。
- 房间类型：普通教室、阶梯教室、临时讨论区。

### 4号教学楼 / 综合教学楼主楼

- 位置：右侧纵向主楼，连接 2/3/5/6 号楼。
- 建议楼层：5 层。
- 房间类型：办公室、连廊、楼梯核心、公共休息区。
- 交互重点：可作为建筑群的中枢入口。

### 5号教学楼

- 位置：中上部横向楼体。
- 建议楼层：4 层。
- 房间类型：多媒体教室、普通教室。

### 6号教学楼

- 位置：顶部横向楼体。
- 建议楼层：4 层。
- 房间类型：大教室、自习教室、考试教室。

## 楼层选择交互设计

### 入口

用户在综合教学楼地图视角中点击楼栋模型：

- Raycaster 命中对应楼栋 mesh 或楼栋分组。
- 选中楼栋描边或发光。
- 镜头从等距俯视平滑切换到平视透视。
- UI 顶部显示：`综合教学楼 / 1号教学楼`。

### 楼层选择界面

布局建议：

- 左侧或顶部：返回按钮、楼栋名称、在线人数。
- 中心：3D 楼层模型。
- 右侧：楼层列表。
- 底部：当前楼层简介与房间数量。

交互状态：

- 默认：所有楼层以正常颜色堆叠展示。
- Hover：目标楼层抬高、变亮、显示浮动标签。
- Selected：目标楼层保持高亮，右侧显示房间列表。
- Click：进入楼层页面。

### 楼层模型显示

第一版采用「爆炸楼层视图」：

```text
5F  ─────────────

4F  ─────────────

3F  ─────────────  ← hover 高亮

2F  ─────────────

1F  ─────────────
```

每层内部只做抽象分区，不做精确教室门牌：

- 大教室：较大矩形区域。
- 普通教室：中等矩形区域。
- 走廊：长条区域。
- 楼梯/电梯：竖向核心块。

## 技术实现规划

### 3D 技术栈

| 层 | 技术 |
|---|---|
| 3D 引擎 | Three.js |
| 模型格式 | glTF Binary `.glb` |
| 模型加载 | `GLTFLoader` |
| 视角控制 | `OrbitControls`，限制旋转范围 |
| 点击检测 | `Raycaster` |
| 楼栋高亮 | mesh 材质 emissive / outline pass / 克隆边线 |
| 楼层展示 | 独立 floor group，点击楼栋后加载或显示 |
| UI 层 | HTML/CSS 覆盖在 Three.js Canvas 上 |

### Blender / MCP 建模流程

当前建模方式：

- 使用 `ahujasid/blender-mcp` 连接 Blender。
- Codex 通过 BlenderMCP socket 执行 Blender Python。
- 生成综合教学楼 GLB 和预览图。

后续新增楼层模型：

```text

  render_teaching_complex_map_angle.py
  create_teaching_floor_models.py
```

## 第一阶段开发拆分

### Milestone 1：综合楼地图可点击

- 加载 `teaching_complex_1_6.glb`。
- 显示等距地图视角。
- 支持拖拽、缩放、轻微旋转。
- 给 1-6 号楼建立可点击区域。
- 点击后识别楼栋 ID。

### Milestone 2：1号教学楼楼层选择

- 先只实现 `1号教学楼`。
- 生成 `teaching_1_floors.glb`。
- 点击 1 号楼后切换到平视透视。
- 展示 1F-4F 爆炸楼层模型。
- 楼层 hover / selected / click 状态完整。

### Milestone 3：2-6号教学楼楼层选择

- 复用 1 号教学楼楼层组件。
- 分别配置楼层数量和房间类型。
- 4 号主楼可设置 5 层，其他楼 4 层。

### Milestone 4：房间与座位

- 点击楼层后展示房间列表或楼层平面。
- 点击房间进入座位图。
- 座位状态：空座、已占、我的座位。
- 接入匿名聊天。

## 匿名身份系统

- 用户进入时随机分配匿名身份，格式为「颜色 + 动物名」。
- 身份仅当次会话有效，离开即消失。
- 头像使用颜色圆形 + 首字。
- 可刷新身份后确认进入。

示例：

```js
{
  name: '蓝色刺猬',
  color: '#1976d2',
  avatarText: '刺'
}
```

## 聊天与共鸣机制

### 聊天范围

| 层级 | 范围 | 第一阶段状态 |
|---|---|---|
| 房间频道 | 当前房间 | Milestone 4 |
| 楼层频道 | 当前楼层 | Milestone 4 后 |
| 综合楼频道 | 综合教学楼建筑群 | 可作为 MVP 简化版 |
| 全校广播 | 全校园 | 第二阶段 |

### 共鸣机制

- 每条消息可点击「共鸣」。
- 共鸣数达到阈值后，消息可从房间频道升级到楼层频道。
- 第一阶段可以先做本地模拟数据。

## UI 风格规范

### 3D 场景

- 地面：青绿色 `#8fd8a9` 附近。
- 道路：浅灰白，边缘可用细线暗示。
- 路径：浅米色。
- 建筑：粉红/暖白墙体，灰蓝屋顶，浅蓝窗户。
- 树：低多边形圆锥树，绿色和少量黄色点缀。

### 界面组件

- 风格：轻量、干净、校园导览感。
- 主色：蓝色 `#1976d2`。
- 背景：暖灰白 `#f5f4f0`。
- 卡片圆角：8px。
- 按钮圆角：4px。
- 字体：Microsoft YaHei / PingFang SC。
- 不使用过度营销式大 Hero，第一屏直接进入可操作场景。
